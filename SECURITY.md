# Security policy

Voxa Cue is a closed educational prototype. It is not approved for public AI
access, unattended hardware use, safety-critical alerts, or production wearable
distribution.

## Reporting a vulnerability

GitHub private vulnerability reporting is not currently enabled for this
repository, and no monitored private security mailbox has been designated. Do
not publish exploit details or secrets in a public issue. Contact the
[`amaar-mc`](https://github.com/amaar-mc) repository owner without sensitive
details and request a private channel first.

Enabling a monitored private reporting channel is required before public
distribution. Once available, a report should include the affected commit,
component, reproduction steps, expected impact, and whether it involves API
credentials, transcript data, Bluetooth actuation, or physical hardware. Never
include real transcripts, credentials, or other people's personal data.

## Supported version

Only the current `main` branch is maintained. The repository's release
checklist and audit describe the controls that must be completed before any
public distribution.
